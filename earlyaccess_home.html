    What are Early Access Builds?

    Early access builds allow developers to prepare for the next SDK release by providing upcoming features and bug fixes prior to general availability.

    Early Access Features

    This Early Access release is focused on our Tags 2.0 (formerly Mattertags) beta.

    Note: Before being able to use Tags 2.0, please contact developer support to be opted into the beta.

    Known Issues

    • Tag.allowAction doesn’t always prevent actions when interacting with the billboard UI component
    • Removing a tag with Tag.remove doesn’t close it billboard if it was open when removing it
    • When editing a tag, editing the text content or attaching media, isn’t immediately rendered and requires the tag’s billboard be closed and reopened to display the updated contents
    • Mattertag Events no longer fire

    Changelog

    June 30, 2022
    • Added Tag.detach to remove attachments from a tag
    • Removed the need to specify the type when calling Tag.registerAttachment
    • Added Tag.registerSandbox which should be used as a replacement for Tag.addSandbox
    • Added Tag.editStem and Mattertag.editStem to control the stem’s length and/or visibility
    • Asset.registerTexture should now be used in place Pointer.registerTexture. Assets from Asset.registerTexture can be used in Pointer.editTexture, Mattertag.editIcon, and Tag.editIcon
    • Fixed an issue where Embed.ly key overrides were not respected
    June 7, 2022
    • Added the Tag namespace.
    • Added Label.data collection to receive Labels data
    • Added Tour.currentStep, Tour.transition, and Tour.state to get information about what the tour is currently doing
    • Added Mode.current and Mode.transition to get information about the current view mode and any transitions between them taking place
    • Added Asset.registerTexture to be used a central place for registering icons like those for Tags or Mattertags
    • Updated internal Three.js from r136 to r139
