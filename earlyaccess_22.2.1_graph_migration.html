Pathfinding

What has changed?

ICollectionAdaptor

  • New: isNeighborOf is a new, now required method
  • Change: neighborsOf and weightBetween are now called with the vertices instead of the data within each vertex

Why these changes?

The inclusion of isNeighborOf allows the IDirectedGraph.watch to be more performant when updating the graph structure.

Calling neighborsOf and weightBetween with the vertex provides more context about the vertex than just the underlying data and allows for more perfomant updates to the graph’s edges.

Migrating from 3.1.55.2-34-ga9934ccd93

In order to migrate from a prior version of the ICollectionAdaptor

Add the ‘isNeighborOf’ method

The additional isNeighborOf method needs to be included in the object passed to IDirectedGraph.watch.

For example with sweeps

graph.watch({
  // ...
  isNeighborOf(src, dst) {
    return src.data.neighbors.includes(dst.id);
  },
  // ...
});

Access the ‘.data’ of the vertices

In the neighorsOf and weightBetween methods, when accessing the src and dst, access src.data and dst.data instead.

For example with sweeps

graph.watch({
  // ...
  neighborsOf(sweepVertex) {
    return sweepVertex.data.neighbors.values();
  },
  weightBetween(sweepSrc, sweepDst) {
    const dx = sweepDst.data.position.x - sweepSrc.data.position.x;
    const dy = sweepDst.data.position.y - sweepSrc.data.position.y;
    const dz = sweepDst.data.position.z - sweepSrc.data.position.z;
    return Math.sqrt(dx ** 2 + dy ** 2 + dz ** 2);
  },
  // ...
});

instead of

graph.watch({
  // ...
  neighborsOf(sweep) {
    return sweep.neighbors.values();
  },
  weightBetween(sweepSrc, sweepDst) {
    const dx = sweepDst.position.x - sweepSrc.position.x;
    const dy = sweepDst.position.y - sweepSrc.position.y;
    const dz = sweepDst.position.z - sweepSrc.position.z;
    return Math.sqrt(dx ** 2 + dy ** 2 + dz ** 2);
  },
  // ...
});
